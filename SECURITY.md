# Security Policy

Please do not disclose vulnerabilities through a public issue.

For a real deployment, report security concerns privately to the repository owner with:

- affected route or component
- reproduction steps
- expected and observed behavior
- potential impact
- suggested remediation, where available

Never attach real vendor documents, credentials, tokens, production logs, or personal data to a report. Public demonstrations must use the controlled records provided by `DemoSeeder` only.

The implementation security model is documented in [docs/SECURITY.md](docs/SECURITY.md).
